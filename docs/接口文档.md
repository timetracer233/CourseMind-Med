# 接口文档

本文档描述本系统各模块的输入输出接口。

## 1. 数据结构

### Textbook

```python
@dataclass
class Textbook:
    filename: str              # 文件名
    status: ParseStatus        # PENDING / PARSING / DONE / FAILED
    chapters: list[Chapter]    # 章节列表
    total_chars: int           # 总字数
    total_pages: int           # 总页数
    error: str                 # 错误信息（如有）
```

### Chapter

```python
@dataclass
class Chapter:
    title: str                 # 章节标题
    page_start: int            # 起始页
    page_end: int              # 结束页
    char_count: int            # 字数
    text: str                  # 正文内容
```

### Chunk

```python
@dataclass
class Chunk:
    chunk_id: str              # 块 ID
    textbook: str              # 教材名
    chapter: str               # 章节名
    page: int                  # 页码
    text: str                  # 正文
```

### KnowledgeNode

```python
@dataclass
class KnowledgeNode:
    name: str                  # 知识点名称
    definition: str            # 定义
    category: str              # 类别（核心概念/机制/方法/现象/结构/概念）
    chapter: str               # 章节
    page: int                  # 页码
    textbook: str              # 教材名
```

### KnowledgeEdge

```python
@dataclass
class KnowledgeEdge:
    source: str                # 源节点名称
    target: str                # 目标节点名称
    relation_type: str          # 关系类型（contains/prerequisite/parallel/applies_to）
    description: str           # 关系描述
```

### IntegrationDecision

```python
@dataclass
class IntegrationDecision:
    decision_id: str           # 决策 ID（如 D001）
    action: DecisionAction     # MERGE / KEEP / REMOVE
    affected_nodes: list[str]  # 涉及的知识点名称列表
    result_node: str            # 结果节点名称
    reason: str                # 决策理由
    confidence: float          # 置信度（0.0-1.0）
    status: DecisionStatus     # ACTIVE / OVERRIDDEN
```

### ParseStatus / DecisionAction / DecisionStatus

```python
class ParseStatus(str, Enum):
    PENDING = "pending"
    PARSING = "parsing"
    DONE = "done"
    FAILED = "failed"

class DecisionAction(str, Enum):
    MERGE = "merge"    # 合并重复知识点
    KEEP = "keep"      # 保留知识点
    REMOVE = "remove"  # 删除知识点

class DecisionStatus(str, Enum):
    ACTIVE = "active"      # 正常决策
    OVERRIDDEN = "overridden"  # 被教师反馈覆盖
```

## 2. 模块接口

### 2.1 parse_file

**文件**：`src/parsers.py`

**签名**：
```python
def parse_file(filepath: str) -> Textbook
```

**说明**：解析单个教材文件，根据扩展名调用对应解析器。

**返回**：`Textbook` 对象，status 为 DONE 或 FAILED。

---

### 2.2 chunk_all

**文件**：`src/chunking.py`

**签名**：
```python
def chunk_all(textbooks: dict[str, Textbook]) -> list[Chunk]
```

**说明**：将所有教材按章节分块，chunk_size=700，overlap=100。

**返回**：`Chunk` 列表。

---

### 2.3 RAGEngine.index

**文件**：`src/rag.py`

**签名**：
```python
class RAGEngine:
    def index(self, chunks: list[Chunk]) -> None
```

**说明**：优先使用 MiniMax embo-01 API 向量化所有 chunk，构建知识库时使用 `type=db`；如果 MiniMax API 不可用，自动回退到 TF-IDF 兜底索引。

---

### 2.4 RAGEngine.search

**签名**：
```python
def search(self, query: str) -> list[tuple[Chunk, float]]
```

**说明**：检索 top-k 最相关 chunk，默认 k=5。

**返回**：`[(Chunk, 相关度分数)]` 列表。

---

### 2.5 RAGEngine.ask

**签名**：
```python
def ask(self, query: str) -> tuple[str, list[dict]]
```

**说明**：检索相关 chunk，调用 DeepSeek 生成答案。

**返回**：
- `str`：答案文本
- `list[dict]`：引用列表 `[{教材, 章节, 页码, 相关度, 原文片段}]`

---

### 2.6 extract_from_textbook

**文件**：`src/knowledge_extractor.py`

**签名**：
```python
def extract_from_textbook(tb: Textbook) -> tuple[list[KnowledgeNode], list[KnowledgeEdge]]
```

**说明**：按章节调用 DeepSeek 抽取知识点和关系。无 API Key 时使用规则兜底。

**返回**：`(节点列表, 边列表)`

---

### 2.7 build_graph_html

**文件**：`src/graph_builder.py`

**签名**：
```python
def build_graph_html(
    nodes: list[KnowledgeNode],
    edges: list[KnowledgeEdge],
    width: str = "100%",
    height: str = "600px",
) -> str
```

**说明**：使用 pyvis 生成可交互 HTML 图谱文件路径。

**返回**：HTML 文件路径，失败返回空字符串。

---

### 2.8 integrate

**文件**：`src/integrator.py`

**签名**：
```python
def integrate(nodes: list[KnowledgeNode]) -> tuple[list[IntegrationDecision], dict]
```

**说明**：基于名称相似度（≥0.82）生成跨教材整合决策。

**返回**：
- `list[IntegrationDecision]`：决策列表
- `dict`：统计信息 `{total_nodes, total_decisions, merge_count, keep_count, remove_count, original_chars, integrated_chars, compression_ratio}`

---

### 2.9 process_feedback

**文件**：`src/feedback.py`

**签名**：
```python
def process_feedback(
    user_input: str,
    decisions: list[IntegrationDecision],
) -> tuple[str, list[IntegrationDecision]]
```

**说明**：解析教师指令，修改对应决策。

**支持的指令**：
| 指令 | 行为 |
|------|------|
| 保留 \<知识点\> | remove → keep，标记 overridden |
| 删除 \<知识点\> | keep → remove，标记 overridden |
| 不要合并 \<A> 和 \<B> | merge → keep，标记 overridden |
| 为什么合并 \<知识点\> | 返回决策理由 |

**返回**：
- `str`：处理解释
- `list[IntegrationDecision]`：更新后决策列表

---

### 2.10 generate_report

**文件**：`src/report_generator.py`

**签名**：
```python
def generate_report(
    textbooks: dict[str, Textbook],
    nodes: list[KnowledgeNode],
    edges: list[KnowledgeEdge],
    decisions: list[IntegrationDecision],
    stats: dict,
    output_dir: str = "report",
) -> str
```

**说明**：汇总系统状态生成 Markdown 报告，写入 `report/整合报告.md`。

**返回**：Markdown 格式报告内容字符串。

---

### 2.11 chat / chat_json

**文件**：`src/llm_client.py`

**签名**：
```python
def chat(messages: list[dict], temperature: float = 0.3, max_tokens: int = 2048) -> str
def chat_json(messages: list[dict], temperature: float = 0.2, max_tokens: int = 2048) -> dict | list | None
```

**说明**：调用 DeepSeek API。`chat` 返回文本，`chat_json` 尝试解析 JSON。

**返回**：文本或 JSON 对象，失败返回空字符串/None。

---

### 2.12 has_api_key

**签名**：
```python
def has_api_key() -> bool
```

**说明**：检查是否配置了 DEEPSEEK_API_KEY。

## 3. 配置文件

**文件**：`src/config.py`

| 变量 | 默认值 | 说明 |
|------|--------|------|
| `DEEPSEEK_API_KEY` | "" | API 密钥 |
| `DEEPSEEK_BASE_URL` | "https://api.deepseek.com" | API 地址 |
| `DEEPSEEK_MODEL` | "deepseek-chat" | 模型名称 |
| `CHUNK_SIZE` | 700 | 分块大小（字符数） |
| `CHUNK_OVERLAP` | 100 | 分块重叠（字符数） |
| `TOP_K` | 5 | 检索返回数量 |
| `MERGE_SIM_THRESHOLD` | 0.82 | 合并相似度阈值 |
| `FAST_MODE_MAX_PAGES` | 60 | 快速模式最大页数 |
| `FAST_MODE_MAX_CHAPTERS` | 8 | 快速模式最大章节数 |

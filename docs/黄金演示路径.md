# 黄金演示路径

本文档用于比赛当天演示和最终测试。目标是保证评委在 3-5 分钟内看到完整价值闭环。

## 1. 演示原则

1. 不演示系统极限能力，只演示稳定路径。
2. 优先使用 1-2 本小型医学教材或从 7 本教材中截取的前若干章节。
3. 所有问题必须提前验证能命中引用来源。
4. 每一步都要能在公网链接上完成。

## 2. 演示数据准备

推荐准备：

- 医学教材样例 A：包含“炎症”“免疫应答”等内容。
- 医学教材样例 B：包含相同或相近知识点，便于展示跨教材合并。

样例要求：

- PDF / MD / TXT 均可。
- 每个文件不宜过大。
- 至少包含 2-3 个清晰章节。
- 至少有一个重复知识点。

如果只能使用正式 7 本教材：

- 启用快速模式。
- 每本只处理前 60 页或前 8 章。
- 演示时只上传 1-2 本。

## 3. 预设问题

根据实际样例最终确认。建议准备 3 个问题：

1. 炎症的定义是什么？
2. 免疫应答包含哪些核心环节？
3. 动作电位和静息电位有什么关系？

每个问题都需要提前记录：

- 预期命中的教材。
- 预期章节。
- 预期页码或 chunk。
- 一句标准答案。

## 4. 标准演示流程

### Step 1：打开公网链接

展示内容：

- 项目名称。
- 上传区。
- Tab 布局。

讲解话术：

> 这是一个面向医学教材测试集的通用教材知识整合智能体，支持上传多本教材后自动解析、建图、去重压缩和带引用问答。

### Step 2：上传教材并解析

操作：

1. 上传 1-2 本小教材。
2. 点击“解析教材”。

期望结果：

- 文件状态为“已完成”。
- 显示教材数量、章节数量、总字数。
- 章节表出现标题、页码、字数。

讲解话术：

> 系统把不同格式教材统一转成结构化章节，后续所有知识抽取和 RAG 都基于这个结构。

### Step 3：构建知识图谱

操作：

1. 点击“抽取知识点/构建图谱”。

期望结果：

- 出现节点和边。
- 图谱可缩放、拖拽。
- 悬停或点击节点可看到定义、章节、来源。

讲解话术：

> 节点代表教材中的核心知识点，边代表前置、包含、并列或应用关系。颜色表示教材来源，大小表示出现频次。

### Step 4：跨教材整合

操作：

1. 点击“执行跨教材整合”。

期望结果：

- 出现 merge / keep / remove 决策表。
- 显示压缩比统计。
- 至少一条决策有清楚理由。

讲解话术：

> 系统使用名称规则和语义相似度识别重复知识点，并给出合并、保留或删除建议，目标是把多本教材压缩到 30% 以内。

### Step 5：RAG 问答

操作：

1. 输入预设问题，如“炎症的定义是什么？”
2. 点击提问。

期望结果：

- 有中文答案。
- 答案下方有引用列表。
- 引用包含教材、章节、页码、相关度。
- 可查看原文 chunk。

讲解话术：

> 这里不是普通聊天，系统只基于上传教材回答，并给出来源，避免医学场景下无依据回答。

### Step 6：教师反馈

操作：

输入：

```text
不要合并 炎症 和 炎症反应
```

或根据实际节点替换名称。

期望结果：

- 对应 merge 决策变为 overridden。
- 聊天窗口返回解释。
- 决策表更新。

讲解话术：

> 教材整合不能完全交给 AI，教师可以覆盖自动决策，系统会保留修改后的方案。

### Step 7：生成整合报告

操作：

1. 点击“生成报告”。

期望结果：

- 页面显示 Markdown 报告。
- `report/整合报告.md` 被更新。
- 报告包含概览、决策摘要、图谱统计、典型案例、教学完整性说明。

讲解话术：

> 最终报告可以作为教研室评估教材重复、缺失和课程整合依据。

## 5. 演示失败兜底

| 失败点 | 兜底话术/操作 |
| --- | --- |
| DeepSeek 慢 | 使用检索式摘要，仍展示引用来源 |
| 图谱未生成 | 展示节点表和边表，说明 HTML 图谱稍后刷新 |
| PDF 解析慢 | 切换快速模式或上传小样例 |
| 魔搭链接异常 | 使用 Gradio share 链接 |
| 教师反馈没匹配 | 使用节点表中的真实名称重试 |

## 6. 最终演示前检查

- [ ] 公网链接可打开。
- [ ] 上传样例文件成功。
- [ ] 预设问题能返回引用。
- [ ] 图谱不是空白。
- [ ] 有至少一条 merge 决策。
- [ ] 压缩比统计正确显示（原始字数 vs 整合后字数 vs 压缩比百分比）
- [ ] 教师反馈能修改决策。
- [ ] 报告能生成。

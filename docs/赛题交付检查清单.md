# 赛题交付检查清单

本文档用于比赛全过程和最后 30 分钟逐项检查。凡是标为“必须”的内容，缺失会直接影响评审。

## 1. 必交项

| 项目 | 状态 | 检查方式 |
| --- | --- | --- |
| GitHub 公开仓库 | 必须 | 未登录状态可打开仓库 |
| 公网部署链接 | 必须 | 浏览器无登录可打开并使用 |
| README.md | 必须 | 包含依赖、安装、配置、运行、使用说明 |
| docs/需求分析.md | 必须 | 覆盖知识粒度、重复判定、压缩比、RAG 分块 |
| docs/系统设计.md | 必须 | 包含架构、数据流、技术选型、接口说明 |
| docs/Agent架构说明.md | 必须 | 有 Mermaid 图、设计论证、取舍分析 |
| report/整合报告.md | 必须 | 包含 5 项报告内容 |
| .gitignore 排除 PDF | 必须 | 包含 `*.pdf` 和 `data/textbooks/` |
| 黄金演示路径 | 必须 | 上传-解析-图谱-整合-RAG-反馈-报告能跑通 |

## 2. P0 功能逐项映射

| 赛题要求 | 项目实现目标 | 检查方式 |
| --- | --- | --- |
| 多格式教材加载 | PDF / Markdown / TXT 上传解析 | 至少 PDF + 一种文本格式成功，最好三种都测 |
| 章节结构识别 | 识别“第X章”等中文章节 | 章节表显示标题、页码、字数 |
| 单本教材知识图谱 | 从章节抽取知识点与关系 | 节点表、边表有数据 |
| 图谱可视化 | pyvis 可交互图谱 | 可缩放、拖拽、点击/悬停 |
| 跨教材整合 | 相似知识点合并 | 至少两本教材生成 merge 决策 |
| 整合决策说明 | 每条决策有原因和置信度 | 决策表包含 reason/confidence |
| 压缩比不超过 30% | 原始字数、整合后字数、压缩比 | 页面指标小于等于 30% |
| RAG 问答 | 基于教材 chunk 回答 | 预设问题能返回答案 |
| 引用来源 | 回答附教材、章节、页码、分数 | 引用表可见并能展开 chunk |
| Agent 架构文档 | docs/Agent架构说明.md | 有 Mermaid 图和取舍论证 |
| 教师多轮反馈 | 简化规则反馈即可 | 输入“不要合并X和Y”后决策变化 |
| Web 页面 | Gradio 单页应用 | 浏览器中可完成黄金演示路径 |
| 整合报告 | report/整合报告.md | 报告由当前数据生成 |
| 开发文档 | README + docs | 按 README 可本地运行 |

## 3. 文档评分检查

文档和代码必须并行，不允许最后 1 小时才开始写文档。

### README

- Python 版本。
- 安装命令。
- 环境变量配置。
- 启动命令。
- 使用流程。
- 部署说明。

### 需求分析

- 知识点粒度。
- 重复判定标准。
- 教学连贯性保障。
- 压缩比计算方式。
- RAG 分块策略。

### 系统设计

- 架构图。
- 数据流。
- 技术选型。
- 内部接口。
- 错误处理。

### Agent 架构说明

- 架构总览。
- Mermaid 架构图。
- 为什么使用轻量多 Agent 编排。
- 为什么不引入复杂框架。
- RAG pipeline。
- Prompt 工程。
- 已知局限与改进。
- 创新点。

### 整合报告

- 整合概览。
- 决策摘要。
- 图谱统计。
- 3-5 个典型案例。
- 教学完整性说明。

## 4. 功能优先级检查

必须做：

- PDF / MD / TXT 解析。
- RAG 带引用。
- 知识点抽取。
- 可交互图谱。
- 跨教材整合决策。
- 压缩比。
- 简化教师反馈。
- 整合报告。
- 公网链接。

明确不做：

- DOCX / Excel。
- Docker。
- 混合检索 / Rerank。
- 多视图图谱。
- 本地大模型。
- 全量大教材深度处理。

## 5. 最后 30 分钟检查流程

1. 确认未提交任何教材 PDF。
2. 本机运行 `python app.py`。
3. 上传黄金演示样例，确认解析。
4. 问答预设问题，确认有引用。
5. 构建图谱，确认图谱不空白。
6. 执行整合，确认有决策和压缩比。
7. 输入教师反馈，确认决策变化。
8. 生成报告。
9. 检查 5 份核心文档不是空模板。
10. 检查 README 启动命令。
11. 推送 GitHub。
12. 打开部署链接测试完整黄金路径。

## 6. 绝对不能犯的错误

- 只有 localhost，没有公网链接。
- GitHub 仓库不是 public。
- 把教材 PDF 推到 GitHub。
- RAG 答案没有引用来源。
- 图谱只是静态图片。
- 没有 `docs/Agent架构说明.md`。
- README 没有运行命令。
- 页面无法上传用户自己的教材。
- 最后 1 小时才开始写文档。
- 为 P1 功能牺牲 P0 闭环。

## 7. 演示话术建议

1. “这个系统面向医学教材测试集，但架构是通用教材知识整合智能体。”
2. “先上传教材，系统自动解析章节。”
3. “然后抽取知识点和关系，生成可交互知识图谱。”
4. “跨教材合并重复概念，给出决策理由和压缩比。”
5. “老师可以用自然语言纠正整合决策。”
6. “RAG 问答只基于教材内容，并给出原文引用。”
